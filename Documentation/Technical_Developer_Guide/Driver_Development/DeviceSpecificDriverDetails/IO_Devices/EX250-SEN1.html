EX250-SEN1 driver document

EX250 SEN1

Introduction

Brief overview of the project

The EX250-SEN1 device from SMC offers an advanced solution for pneumatic control systems, providing seamless integration with EtherNet/IP and versatile connectivity options for optimized performance. With its application across various industries including material handling and food processing, it enhances efficiency and reliability in automated production environments.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the EX250-SEN1 device. The driver will enable communication between the device with the help of third party python library,pycomm3. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the EX250-SEN1 device.

The primary objective of the document is to provide a comprehensive guide for developed drivers.

Scope and objectives

  1. Technical Information: Provide comprehensive technical documentation regarding the EX250-SEN1 device, including device specifications, and capabilities and compile manuals, reference materials to facilitate understanding and implementation by developers and engineers.
  2. Driver Development and Implementation: Create robust device drivers and software interfaces compatible with multiple platforms.Ensure smooth integration of the EX250-SEN1 with industrial systems like PLCs.
  3. Testing and Debugging: Identify and resolve software bugs, compatibility issues, and performance bottlenecks through systematic debugging processes and conduct rigorous testing procedures to validate the functionality and reliability of the device and its drivers.
  4. Documentation and Support: Create comprehensive documentation covering installation guides, programming APIs, troubleshooting procedures, and best practices. Maintain up-to-date documentation to reflect any changes, updates, or enhancements to the device and its software ecosystem.

Device Overview

Description of the device for which the driver is being developed

The EX250-SEN1 is an advanced pneumatic control system developed by SMC, a leading manufacturer in the valve products line. It streamlines operations through reduced wiring complexity and seamless integration with EtherNet/IP, facilitating efficient communication between input and output equipment.

  1. EtherNet/IP Connectivity: Enables streamlined communication between the input and output equipment, reducing wiring complexity.
  2. SI Unit Compatibility: Communicates with input and output equipment via the SI Unit, facilitating easy integration.
  3. Expandable Inputs and Outputs: Supports connection of up to 32 Inputs and 32 Outputs, allowing for versatile system configurations.
  4. Power Block Integration: Accommodates Power Block connection for expanded functionality, with a maximum output point of 24.

Applications

  1. Pneumatic Control Systems: Ideal for managing and controlling the flow of compressed air/water in pneumatic control systems, ensuring precise operation and efficient resource utilization.
  2. Material Handling Systems: Used in controlling conveyor belts, sorting mechanisms, and other equipment involved in material handling processes, optimizing productivity and throughput.
  3. Food and Beverage Processing: Applied in food and beverage processing environments to regulate processes such as filling, packaging, and sorting within automated production lines, ensuring consistency and quality.

Library Overview

pycomm3 is a Python library designed for communication with Allen-Bradley PLCs (Programmable Logic Controllers) using the Ethernet/IP protocol. It allows developers to interface with Rockwell Automation PLCs like CompactLogix, ControlLogix, and Micro800 series. The library is open-source and provides a modern, lightweight alternative for PLC communication tasks.

Getting Started

The package can be installed from PyPI using pip:

pip install pycomm3 or python -m pip install pycomm3

Implementation of pycomm3 library

The Logix5000 Controller Data Access Manual, available from the Rockwell Developer How-to Guides, was used to implement the Ethernet/IP features in this library. Features like reading tags/arrays, writing tags/arrays, getting the tag list are all implemented based on the Data Access Manual. The Rockwell KB Article CIP Messages References 748424 lists many useful KB Articles for using the MSG instruction to perform various Ethernet/IP services. The Rockwell Knowledge Base Article 23341 was used to implement feature for getting the program name of the controller. Article 28917 was used for collecting other controller information.

DRIVER DEVELOPMENT

Introduction to driver development for the device

The CIPDriver is the foundational class for all drivers within the pycomm3 library. It serves as a base for implementing communication with devices using the Common Industrial Protocol (CIP).

The CIPDriver provides to class methods for discovering and identifying devices. And because they are class methods, they can be used without creating an instance of a driver first. The CIPDriver.discover() method will broadcast a request for all devices on the network to identify themselves. This is similar to how the RSLinx Ethernet/IP driver works. It returns a list of dictionaries, where each dict is the Identity Object of the device.

Limitations of the driver

  • The EX250-SEN1 device's capacity for handling inputs and outputs may restrict scalability in larger systems, hindering applications with extensive I/O requirements.
  • Compatibility issues with specific hardware, software, or communication protocols may limit interoperability with other devices or systems.
  • Integrating the device into existing workflows could be complex, especially for users with limited technical expertise or resources.
  • Depending on processing power, it may struggle with high-speed or real-time control tasks, causing performance bottlenecks in demanding environments.

TROUBLESHOOTING TECHNIQUES

Trouble No.1
Trouble Possible Cause Investigation method of cause Remedy
Only the solenoid valve LED lights up Solenoid valve failure Check the troubleshooting for the solenoid valve Correct the trouble of the solenoid valve
Trouble No.2
Trouble Possible Cause Investigation method of cause Remedy
The outputs that are beyond the maximum point number do not operate Inadequate total number of outputs from the solenoid valves and Output block connected with the SI unit Check if the total number of outputs is 32 or less (24 for the solenoid valve VQC series) Eliminate extra unused outputs from the manifold to ensure the number of outputs is 32 or less
Trouble No.3
Trouble Possible Cause Investigation method of cause Remedy
Only the solenoid valve/Output block LEDs do not light up Incorrect wiring for the power supply for solenoid valve and the power supply for output Check that the power supply cable for the solenoid valve and output is not broken, and that the connection between the power supply cable and connector has not loosened Review the connection condition of the power supply cable (If the cable is broken, replace it with a new one)
Check that there is no incorrect wiring of the power supply cable Review the wiring condition of the power supply cable
Failure of the power supply for the solenoid valve and output Check the supply voltage to the power supply for the solenoid valve and output Supply 24 VDC +10fo %/the 5% to the power supply for solenoid valve and output
Incorrect connection between the SI unit and the solenoid valve and/or output block Check that the bolt joining the SI unit with the solenoid valve and Output block has not loosened Tighten the bolts by hand so that there is no gap between the SI unit and the solenoid valve and/or Output block. Tighten at the specified torque (Tightening torque: 0.6 Nm)
Check if the Output block (for high wattage and load and low wattage and load) and power block are mounted in the right position Review the position of the Output block and Power block
Intrusion of liquids such as water Check that unused connectors of the Output block and Power block are plugged with waterproof caps If liquid such as water has got into the Output block and/or Power block, replace the Output block and Power block with new ones

GLOSSARY

  • EtherNet/IP: A widely used industrial network protocol that allows communication between devices on an Ethernet network, often used for real-time control and monitoring in industrial automation systems.
  • SI Unit: Refers to the System Interface Unit, a component that facilitates communication between the ex250-sen1 device and input/output equipment, typically used in industrial control systems.
  • Input Blocks: Components used to interface input signals with the ex250-sen1 device, allowing it to receive signals from sensors, switches, or other input devices.
  • Output Blocks: Components used to interface output signals with the ex250-sen1 device, enabling it to control actuators, valves, or other output devices.
  • Valve Manifolds: Modular assemblies that combine multiple valves into a single unit, often used in pneumatic systems to control the flow of air or other gases.
  • Power Block: A component that provides power to the ex250-sen1 device and connected equipment, typically used to expand the device's capabilities or provide additional output points.
  • Pneumatic Control Systems: Systems that use compressed air or gas to control the movement or operation of mechanical devices, often used in industrial automation for tasks such as actuation and manipulation.
