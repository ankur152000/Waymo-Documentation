Denso 4-axis and 6-axis Driver Development Guide

Denso6 Axis Robot

Introduction

Brief overview of the project

This project focuses on developing drivers for Denso 6 axis (HS-45352) and 4 axis(VP-6242) robot to be utilized in automation applications, with a particular emphasis on integrating them with a user interface (UI) application for future use. The project aims to create software components that enable seamless communication between Denso Robot and the UI application, allowing for intuitive control and monitoring of the motion systems. Through this integration, the project seeks to enhance the usability and functionality of Denso within automated processes, contributing to increased efficiency and productivity in various industrial settings.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the Denso robot device. The driver will enable communication between the Robot and external software applications, allowing to movement, control speed, direction, Returning Home and many more. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the Denso robot.

The primary objective of the document is to provide a comprehensive guide for developed drivers for Denso Robo used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

Scope and objectives

  1. Driver Development: The document focuses on the development of a driver for Denso Robot using industry-standard protocols.
  2. Technical Information: It includes technical specifications of the device, hardware and software requirements, and information about the Software Development Kit (SDK) provided for the device.
  3. SDK Overview: Introduction to ORiN2 SDK Interface and its Installation.
  4. Driver Implementation: It covers the development process, including the choice of programming language, information on APIs, how to use APIs, limitations of the driver, precautions to take before execution, and the actual implementation of the driver.
  5. Testing and Debugging: The scope extends to testing and debugging procedures to ensure the reliability and functionality of the driver for real-time application
  6. Resources and Links: The document serves as a comprehensive resource for developers, offering detailed information on the Denso 6-axis Robot device Manuals, Installation Links required for software and drivers, and reference videos for setup, configuration etc.

Overall, the aim is to facilitate the successful development and implementation of the Denso robot, ensuring it meets the required standards and serves its intended purpose effectively.

Device Overview and SDK Installation

Description of the device for which the driver is being developed

The Denso VP-6242 is a compact 6-axis articulated robot designed for high-precision tasks such as assembly, handling, and inspection. With a total arm length of 420 mm, the VP-6242 has segments of 210 mm for each of the first two joints and an additional 75 mm offset on Joint 3, allowing for a versatile range of motion. Equipped with absolute encoders, this robot eliminates the need for homing at startup, enhancing positional accuracy, and its AC servomotors provide smooth, precise control with brakes on each joint for added safety. Capable of handling a maximum payload of 2.5 kg, it is optimized for speed with a high composite speed and short cycle time, making it suitable for applications requiring consistent accuracy. The robot's built-in user air piping and signal lines simplify integration with pneumatic tools, sensors, and other end-effectors, while its compact footprint allows for easy installation in confined spaces. Its low noise operation makes it suitable for quieter environments, and its robust design supports repetitive tasks across various industries, including electronics, automotive, and medical.

The DENSO 4-axis Robot (HS-45352) is a high-speed, compact SCARA robot designed for precision assembly and material handling tasks. Controlled via the RC7 controller, it offers reliable performance with flexible programming support, making it ideal for automation in electronics, automotive, and packaging industries

SDK OVERVIEW

ORiN2 is a middleware that offers a standard interface of various resources, like various Factory Automation (FA) equipment and databases, etc. including robots. By using ORiN, applications can be developed without depending on the manufacturer or the model type.

ORiN2 is composed of the following three basic technologies

• CAO (Controller Access Object)

• CRD (Controller Resource Definition)

• CAP (Controller Access Protocol)

CAO is "Standard program interface" to access robots controller from client applications. CAO is developed based on the distributed object technology, and it is applied not only to the industrial robots but also PLC (Programmable Logic Controller) and the NC machine tool, etc. CAO has expanded the range of its application.

CRD is "Standard Data Schema" to share robot controller's resource information without depending on the robot manufacture, and uses XML files. CAO is a common API, and CRD is a common data expression. CRD is developed based on the XML technology, and can express various types of data, which differ in each robot manufacturer, by single CRD schema. Just like CAO, application field of the CRD basic data schema is not limited to the robot, but "Production information" etc. also can be expressed.

CAP is "Communication protocol for the Internet" to access CAO provider over the Internet. CAP is developed based on the SOAP (Simple Object Access Protocol) technology, and offers the function to access remote controllers without forcing ORiN application developer considering the Internet.

The ORiN Software Suite includes a powerful and versatile Software Development Kit (SDK) designed to accelerate application development on Windows platforms, including both desktop and embedded systems. ORiN is a licensed SDK and provides a well-structured, intuitive API, allowing developers to control and configure DENSO robots and motors with minimal code.

Features Of ORIN

ORiN SDK is a software tool kit used to develop an application program or provider based on ORiN2 specification.

  1. It provides a standard communication interface for robots as well as various peripherals and databases.
  2. The superior expandability of ORiN®2 supports not only industrial robots, but a variety of devices (including PLC, CNC machine tools, bar code readers and RFID) to enable application development that is independent of manufacturer or model.
  3. Supports multiple programming languages including C++, C, and more.
  4. Enables fast integration with Smart Motors and multi-axis systems.
  5. ORiN®2 SDK is mounted with a variety of functions (including a CAO engine, test program, sample program and skeleton provider auto generate tool) to support development.
  6. Covers a variety of use cases including:
    1. Homing
    2. Speed and Direction Control
    3. Program Execution
    4. Multi-motor coordination
    5. Real-time monitoring
    6. Error handling and diagnostics

    With just a few lines of code, users can initiate motor control, perform configurations, and monitor performance—making it suitable for both beginner and advanced developers.

    Parameter Description

    ip_address Specifies the IP address of the device.

    port_address Denotes the specific port address for the device on the communication interface.
    device_id Unique identifier for the device, used to distinguish it from other devices in the system.
    file_path The path to the file, which may include configuration, program, or data files for the device.
    device_model Represents the model number or type of the device being communicated with.
    device_alias A user-defined alias for the device, providing a more descriptive or convenient reference.
    axis Specifies the axis number (e.g., X, Y, Z) on which the command or operation will be performed.

Limitations of the driver

One limitation of the Denso driver may lie in its compatibility with certain operating systems or hardware configurations. While efforts are made to ensure broad compatibility, there may be instances where the driver does not fully function or is not supported on all platforms. Additionally, given the complexity of robotic systems and the variety of tasks they may perform, the Denso driver may not cover every possible use case or scenario. Certain advanced functionalities or specialized operations may not be fully supported by the driver. Furthermore, as with any software development project, the Denso driver may encounter bugs, glitches, or compatibility issues with other software components. These issues could potentially impact the reliability or performance of the driver in certain situations.
Compatibility:The driver will work only the WipCaps should be run in background or else it will throw license error.
Integration:Integrating the driver with other systems or software may pose challenges, especially if there are compatibility issues or dependencies.

Precautions to take before executing the driver

  • Ensure that no error should be in the Pendant.
  • Ensure the program is in task list, while doing Select Program or Run Program.
  • Ensure to provide correct value in PTP movement.
  • Ensure to provide velocity and acceleration with the limit to avoid physical damage of the hardware.

    When developing a Denso driver, it's helpful to have a glossary of terms to ensure clarity and consistency in communication. Here's a glossary of terms commonly used in Denso driver development:

    • Denso Driver: Software module or library responsible for interfacing with Denso robotic systems, facilitating communication and control.
    • Robot Controller: Device or software component that manages the operation of a Denso robot, including motion control, program execution, and I/O handling.
    • API (Application Programming Interface): Set of protocols, tools, and definitions that allow software applications to communicate and interact with the Denso driver.
    • Command: Instruction or directive sent to a Denso robot through the driver to perform specific actions such as movement, I/O control, or program execution.
    • Response: Feedback or acknowledgment received from the Denso robot after executing a command, indicating the success or failure of the operation and providing relevant information or data.
    • Motion Control: Functionality within the Denso driver responsible for controlling the movement of the robot, including linear motion, joint motion, and velocity control.
    • I/O (Input/Output): Interface for interacting with external devices or sensors connected to the Denso robot, allowing the driver to read inputs and control outputs.
    • Program Execution: Process of running predefined motion sequences or routines on the Denso robot, typically stored in the robot controller's memory or loaded from external sources.
    • Configuration: Settings and parameters that can be adjusted or customized within the Denso driver to tailor its behavior to specific requirements or environments.
    • Error Handling: Mechanism for detecting, reporting, and managing errors or exceptions that occur during interaction with the Denso robot, ensuring robustness and reliability.
    • Coordinate System: Reference frame used to specify the position and orientation of the Denso robot's end-effector or individual joints, facilitating precise motion control and trajectory planning.
    • Velocity: Speed at which the Denso robot's end-effector or joints move during motion, typically measured in units per second (e.g., millimeters per second or degrees per second).
    • Acceleration: Rate of change of velocity over time, determining how quickly the Denso robot can accelerate or decelerate during motion, influencing performance and efficiency.
    • Positioning: Process of accurately locating and orienting the Denso robot's end-effector or individual joints at specific coordinates within its workspace, essential for precise manipulation tasks.
    • Feedback: Information provided by sensors or encoders on the Denso robot, used to monitor its state, verify motion accuracy, and adjust control parameters in real-time.
    • Interpolation: Algorithm used by the Denso driver to generate smooth and continuous motion trajectories between discrete points, ensuring seamless movement and avoiding jerky motion.
