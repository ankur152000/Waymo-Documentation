Prosilica GT6600 driver document

Prosilica GT6600-camera

Introduction

Brief overview of the driver

This project focuses on developing drivers for Prosilica GT6600 Camera to be utilized in automation applications, with a particular emphasis on integrating them with a user interface (UI) application for future use. The project aims to create software components that enable seamless communication between Prosilica GT6600 Camera and the UI application, allowing for the camera settings, capture of images, and streaming of data. Through this integration, the project seeks to enhance the usability and functionality of Prosilica GT6600 Camera within automated processes, contributing to increased efficiency and productivity in various industrial settings.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the Allied Vision Camera – Prosilica GT6600 device. The driver will enable communication between the Camera and external software application(pylon viewer), allowing to capture an Image, Live Streaming change camera settings like pixel format, exposure time, gain and many more. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the Allied Vision Camera – Prosilica GT6600 device. The primary objective of the document is to provide a comprehensive guide for developed drivers for Allied Vision Camera – Prosilica GT6600 device used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

The primary objective of the document is to provide a comprehensive guide for developed drivers for Prosilica.

Device Overview

Description of the device for which the driver is being developed

The Prosilica GT6600 stands out as a reliable and rugged camera solution, offering high-performance imaging capabilities for demanding outdoor applications. Its advanced features, robust construction, and wide range of applications make it a valuable tool in various industries requiring high-quality imaging in challenging environments. Key features and characteristics of the Prosilica device include:

  • Sensor: The camera features a Truesense KAI-29050 sensor, providing high-resolution imaging capability.
  • Connectivity: With Power over Ethernet (PoE) support and Ethernet surge suppression, the GT6600 offers reliable connectivity and power supply, even in harsh conditions.
  • Smart Features: The GT6600 includes a range of smart features such as auto exposure, auto gain, auto white balance, flexible binning, and region of interest (ROI) readout, enhancing its usability and adaptability to different imaging scenarios.
  • Global Shutter: Equipped with a global shutter (digital shutter), the camera ensures distortion-free imaging of fast-moving objects.
  • Models: Available in both mono (GT6600) and color (GT6600C) versions, the camera offers flexibility to suit various imaging requirements.
  • Applications: The Prosilica GT6600 finds applications in outdoor imaging, traffic imaging/ITS, public security and surveillance, industrial inspection, machine vision, as well as military and space applications, highlighting its versatility and adaptability across different industries.

Vimba C++ API is an object-oriented API that enables programmers to interact with Allied Vision cameras independent of the interface. It utilizes GenICam transport layer modules to connect to the various camera interfaces and is therefore generic. Features and capabilities of the SDK.

DRIVER DEVELOPMENT

Introduction to driver development for the device

The Prosilica GT6000 camera offers high-performance imaging capabilities for various industrial applications. To harness its full potential, developers can create custom drivers to interface with the camera and control its functionality.

Supported Languages : The Vimba X SDK supports multiple programming languages, including C, C++, and Python, allowing developers to choose the language they are most comfortable with for driver development.

Installation Location : Upon installation, the Vimba X SDK is typically located at:

C:\Users\Public\Documents\Allied Vision\Vimba X

Vimba X Directory Structure : The SDK includes example projects for C, C++, and Python, providing developers with a starting point for their driver development efforts.

VimbaX Shortcut : A shortcut named "VimbaX" is created during installation, leading to the directory:

C:\Program Files\Allied Vision\Vimba X

  • This directory contains essential files and resources for driver development.
  • C:\Program Files\Allied Vision\Vimba X\api\bin houses DLLs required for interfacing with the camera.
  • C:\Program Files\Allied Vision\Vimba X\bin contains additional DLLs and supporting files necessary for driver development and camera control.

Choice of programming language (Python or C++)

For developing the driver we have used C++.

The best API for our purpose

Vimba provides four APIs:

  • The Python API is ideal for quick prototyping. We also recommend this API for an easy start with machine vision or embedded vision applications. For best performance and deterministic behavior, the C and C++ APIs are a better choice.
  • The C API is easy-to-use, but requires more lines of code than the Python API. It can also be used as API for C++ applications. The C API is also recommended to easily migrate from PVAPI to Vimba.
  • The C++ API is designed as a highly efficient and sophisticated API for advanced object-oriented programming including shared pointers, the STL (Standard Template Library), and interface classes. If you prefer an API with less design patterns, we recommend the Vimba C API.
  • The .NET API supports all .NET languages, for example, C#, C++/CLI, or Visual Basic .NET. Its general concept is similar to the C++ API.

    GLOSSARY

    Glossary of Terms for Prosilica GT6000 Camera:

    • Resolution: The level of detail in an image, typically measured in pixels horizontally and vertically. Higher resolution provides greater clarity and detail in captured images.
    • Frame Rate: The number of frames captured by the camera per second. Higher frame rates allow for smoother motion capture, essential for applications requiring real-time monitoring or high-speed imaging.
    • Sensor Size: The physical dimensions of the camera's image sensor, often expressed in inches or millimeters. Larger sensors generally capture more light and offer better image quality, particularly in low-light conditions.
    • Dynamic Range: The range of light intensity levels that a camera can capture, from the darkest shadows to the brightest highlights, without losing detail. A wider dynamic range enables the camera to capture scenes with high contrast more effectively.
    • Pixel Size: The size of individual pixels on the camera sensor, typically measured in micrometers (µm). Larger pixel sizes can capture more light, improving low-light performance and reducing noise in images.
    • Bit Depth: The number of bits used to represent each pixel's color or grayscale value. Higher bit depths provide greater color fidelity and dynamic range but require more storage space and processing power.
    • Interface: The method by which the camera communicates with external devices, such as a computer or controller. Common interfaces include Gigabit Ethernet, USB, and Camera Link.
    • Triggering: The process of initiating image capture based on external events or signals, such as a sensor input, timer, or software command. Precise triggering is crucial for synchronizing multiple cameras or capturing events at specific moments.
    • ROI (Region of Interest): A defined area within the camera's field of view where image capture and processing are focused. Limiting image processing to the ROI can improve efficiency and reduce data transfer and processing requirements.
    • Exposure Time: The duration for which the camera's sensor is exposed to light during image capture, typically measured in milliseconds (ms) or microseconds (µs). Adjusting exposure time controls the amount of light reaching the sensor, affecting image brightness and clarity.
    • Gain: An amplification factor applied to the camera's sensor signal to increase image brightness in low-light conditions. However, excessive gain can introduce noise and reduce image quality.
    • White Balance: The adjustment of color balance in an image to accurately represent true colors under different lighting conditions. White balance settings compensate for variations in light color temperature, ensuring accurate color reproduction.
    • Shutter Mode: The camera's operating mode for controlling exposure time and capturing images. Common shutter modes include rolling shutter, global shutter, and electronic rolling shutter, each suitable for different applications and motion conditions.
    • Image Processing: The manipulation and enhancement of captured images to improve quality, extract relevant information, or perform analysis. Image processing functions may include noise reduction, sharpening, color correction, and object detection.
    • SDK (Software Development Kit): A set of tools, libraries, and documentation provided by the camera manufacturer to facilitate software development for controlling and interacting with the camera. SDKs typically support multiple programming languages and offer APIs for accessing camera features and data.
