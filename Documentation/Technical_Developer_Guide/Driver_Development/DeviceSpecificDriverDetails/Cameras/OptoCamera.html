Opto Camera driver document

Opto Camera

Introduction

Brief overview of the project

This project focuses on developing drivers for Opto Camera (coe-050-m-poe-050ir-c) to be utilized in automation applications, with a particular emphasis on integrating them with a user interface (UI) application for future use. The project aims to create software components that enable seamless communication between Opto Camera and the UI application, allowing for the camera settings, capture of images, and streaming of data. Through this integration, the project seeks to enhance the usability and functionality of Opto Camera within automated processes, contributing to increased efficiency and productivity in various industrial settings.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the opto Camera (coe-050-m-poe-050ir-c). The driver will enable communication between the Camera and external software application (OECS Camera Suite), allowing to capture an Image, Live Streaming, change camera settings like pixel format, exposure time, gain and many more.

It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the opto Camera.

The primary objective of the document is to provide a comprehensive guide for developed drivers for Opto Camera (coe-050-m-poe-050ir-c) used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

Scope and objectives

  1. Driver Development: The document focuses on the development of a driver for Opto Camera using industry-standard protocols
  2. Technical Information: It includes technical specifications of the device, hardware and software requirements, and information about the Software Development Kit (SDK) provided for the device.
  3. SDK Overview: Introduction to OECS Software Suite and its Installation.
  4. Driver Implementation: It covers the development process, including the choice of programming language, information on APIs, how to use APIs, limitations of the driver, precautions to take before execution, and the actual implementation of the driver.
  5. Testing and Debugging: The Scope extends to testing and debugging procedures to ensure the reliability and functionality of the driver for real-time application.
  6. Resources and Links: he document serves as a comprehensive resource for developers, offering detailed information on the Opto Camera device Manuals, Installation Links required for software and drivers, and reference videos for setup, configuration etc.
    7. Overall, the aim is to facilitate the successful development and implementation of the opto Camera, ensuring it meets the required standards and serves its intended purpose effectively.

Device Overview

The COE-G series includes Gigabit Ethernet cameras equipped with the latest sensors, ranging from high speed VGA to the latest 12MP SONY Pregius sensor, which deliver GigE connectivity with high frame rate.

Key Advantages of Opto Camera

  1. High quality sensors: New SONY Pregius CMOS Global shutter sensors provide high quality images.
  2. GigEVision® protocol & GenICam® standard : Standard vision SDK platform for easy integration in existing software.
  3. Full GenICam® compliant: easy to integrate GenICam® compliant SDK package provides more flexibility to Vision Systems
  4. GigE PoE compliance: With the COE-G cameras, you do not need separate cables to transfer the information to the computer and provide power to the camera.
  5. 120 MB RAM and Frame Rate up to 300fps High frame rate ideal for high speed applications. The internal memory up to 120MB guarantees no image loss and enables useful features like Record / Playback and sequence recordings.

Description of the device for which the driver is being developed

The Opto Engineering COE-050-M-POE-050IR-C is a compact, high-performance camera designed for industrial applications. It features a 5 MP sensor with Power over Ethernet (PoE) support, ensuring efficient power and data transmission through a single cable. The camera is equipped with an IR filter, making it suitable for precise imaging in environments with controlled lighting or infrared requirements. Its robust design ensures reliability in demanding conditions, making it ideal for machine vision, automation, and inspection tasks.

Software OECS

Software Overview and Requirements - OECS

The OECS Camera Software Suite is a collection of drivers and tools for operating any OECS camera with Windows. OECS is a graphical environment which is powerful, flexible, easy to use, and are designed to quickly develop and run new vision applications without the need to write program code. Whether you are looking for a development environment suitable for integrators/automation engineers familiar with data-flow programming or software applications designed for users with no machine vision experience or any programming knowledge. Making changes to an existing camera device in your application essentially becomes a plug and play process.

Features Of OECS

  • OECS is compatible with standard protocols such as GigEVision and USB3Vision (currently, GigEVision, USB3Vision, and CameraLink protocols are supported), which contains drive program and API functions. This SDK is used to connect and control machine vision cameras to realize the functions of image acquisition, parameter configuration, and image post-process.
  • The USB 3.0 interface provides the 3 Gbps bandwidth
  • Supports AEC (automatic exposure control), LUT, Gamma Correction, etc..
  • The color camera adopts the color interpolation algorithm to reproduce colors faithfully. It also supports automatic white balance (AWB).
  • Compatible with USB3.0 Vision Protocol, and third-party software.

Hardware and software requirements

Hardware / System Requirements:

  • Windows 10 (64 bit) or Windows 11 (64 bit)
  • PC with interface GigE
  • Intel Core i5 or higher processor
  • 8 GB RAM
  • Power supply for the camera- Power Over Ethernet
  • 5 GB of available hard disk space
  • 5400 RPM hard drive
  • DirectX 9-capable video card running at 1024 x 768 or higher display resolution

Software Requirements:

  • OECS Software Suite
  • Microsoft Visual Studio for C++ and python IDE for development
  • Additional software or libraries required by the SDK or application

Additional Requirements

    This version of Microsoft Visual Studio works best with Internet Explorer 10 or higher. If Internet Explorer 10 is not installed on your computer, some features might not work as expected

GLOSSARY

  • Driver:A software component that enables communication between the operating system and a hardware device, in this case, a camera.
  • SDK (Software Development Kit): A set of tools, libraries, and documentation that developers use to create software applications for a specific platform or framework.
  • API (Application Programming Interface):A set of rules and protocols that allow different software applications to communicate with each other.
  • Firmware:: Software that is embedded into a hardware device, such as a camera, to control its operation.
  • Resolution: The number of pixels in an image, typically measured in megapixels (MP), that determines the level of detail and clarity in the image.
  • Frame Rate: The number of frames or images displayed or processed per second, often measured in frames per second (fps).
  • Gain: Gain in the context of a camera refers to the amplification of the signal from the camera sensor. However, increasing the gain too much can introduce noise into the image, reducing its quality. Gain is usually measured in decibels (dB).
  • Exposure Time: The amount of light that reaches the camera sensor, controlled by settings such as shutter speed, and aperture.
  • Pixel: The smallest unit of a digital image, representing a single point of color.
  • Buffer: A temporary storage area in memory used to hold data, such as images, before it is processed or transferred.
  • Streaming: The process of continuously transmitting data, such as video, over a network in real-time.
