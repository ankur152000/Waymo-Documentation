Barcode Reader (DM262X and DM282X) Driver Development Guide

Introduction

Brief overview of the project

This project focuses on developing drivers for Barcode Reader (DM262X and DM282X) to be utilized in automation applications, with a particular emphasis on integrating them with a user interface (UI) application for future use. The project aims to create software components that enable seamless communication between barcode reader and the UI application, allowing for the, capture of images, and acquiring the barcode data. Through this integration, the project seeks to enhance the usability and functionality of barcode reader within automated processes, contributing to increased efficiency and productivity in various industrial settings.

Purpose of the document

The purpose of this document is to provide a detailed guide for developing a driver for the Barcode Reader (DM262X and DM282X). The driver will enable communication between the device and external software application (Data man tool), allowing to capture an Image, Acquire barcode and many more. It covers the software development process, including setting up the development environment, implementing the driver, testing and debugging, and integrating the driver with the Barcode Reader.

The primary objective of the document is to provide a comprehensive guide for developed drivers for Barcode Reader (DM262X and DM282X) used in automation using a UI application. This includes detailed instructions, software installation guide, a device description with its features, and best practices to ensure the successful development and integration of drivers with the devices.

Scope and objectives

  1. Driver Development: The document focuses on the development of a driver for Barcode Reader using industry-standard protocols.
  2. Technical Information: It includes technical specifications of the device, hardware and software requirements, and information about the Software Development Kit (SDK) provided for the device.
  3. SDK Overview: Introduction to Data Man tool and its Installation.
  4. Driver Implementation: It covers the development process, including the choice of programming language, information on APIs, how to use APIs, limitations of the driver, precautions to take before execution, and the actual implementation of the driver.
  5. Testing and Debugging: The scope extends to testing and debugging procedures to ensure the reliability and functionality of the driver for real-time application
  6. Documentation and Support: Lastly, the document includes information on documentation standards and support resources available for developers.
    7.  Overall, the aim is to facilitate the successful development and implementation of the Barcode Reader, ensuring it meets the required standards and serves its intended purpose effectively.

Device Overview

Description of the device for which the driver is being developed

The Cognex DataMan DM282x and DM262x is fixed -mount barcode reader, specially designed for tough manufacturing and logistics applications. It features premium decoding technology with modular options for both hardware and software.

Some of the applications of the Barcode Reader includes:
  • Manutfacturing and Prodiction lines
  • Logistics and supply chain
  • Retail and E-Commerce
  • Healthcar and Pharmaceuticals
  • Agriculture and Farming

Software Requirement

The Cognex DataMan software ecosystem provides robust tools for configuring, testing, and deploying DataMan industrial barcode readers. It allows users to optimize barcode reading performance, integrate with existing systems, and monitor devices in real-time.

  • Microsoft Visual Studio for C# for development
  • Additional software or libraries required by the SDK or application

Software Overview

The Cognex DataMan SDK is a software development kit designed to interact with Cognex DataMan barcode readers.The SDK provides tools, libraries, and APIs to integrate the functionality of DataMan device into custom applications, allowing developers to configure control, and retrive data from DataMan barcode readers programmatically.

Driver Development and SDK Installation

Introduction to driver development for the device

The driver development consists of three key components:

  • barcode.cs

    • barcode.cs plays a crucial role in establishing communication between the software and Barcode Reader devices. It provides APIs to connect to the device, send and receive packets, and encode/decode data according to the Barcode Reader protocol. This code enables the minimum functionalities of the API.

gRPC API’s Description for Barcode Reader Device

  1. GetDeviceDetails

    Description for GetDeviceDetails API:Retrieves detailed information about the connected Barcode Reader, including its configuration and status.

    RPC Command : rpc GetDeviceDetails(.generic. CommandRequest) returns (.generic.GenericDeviceModel) {}

    Request: message CommandRequest {
    string device_alias = 1;
    }
    Parameter Description
    device_alias Unique alias of the device
    Response:
    message GenericDeviceModel{
    string instance_id = 1;
    string device_alias = 2;
    DeviceType device_type = 3;
    DeviceModel device_model_name = 4;
    optional DeviceStatus device_status = 5;
    string ipaddr = 6;
    string mac_addr = 7;
    google.protobuf.Value conn_handle=8;
    string port_name = 9;
    int64 port_address = 10;
    string file_path =11;
    CommandResponse cmd_response = 12;
    }
    Parameter Description
    instance_id New instance for the device in the enumeration
    device_alias Unique identifier (alphanumeric) for every device in the enumeration
    device_type Indicates the type of device like Motion stage, Safety Controller etc (defined below in the DeviceType enum)
    device_model_name Indicate the model name for each device (define below in the DeviceModel enum)
    device_status Indicate whether the device is online , offline or not reachable (define below in the device status enum)
    ipaddr Indicate the ipaddress of the device
    mac_addr Indicate the mac address of the device
    Conn_handle Handle object to keep the device open
    port_name Port name
    port_address Port address
    cmd_response Indicates command response that describes API response code and its description (Defined below in CommandResponse)
    file_path Indicates file name with current its current location(file path)
    enum DeviceType{
    NONE = 0;
    SENSOR_3D = 1;
    MOTION_STAGE = 2;
    CAMERA = 3;
    IO_MODULE = 4;
    SAFETY_CONTROLLER = 5;
    ANIMATIC_MOTOR=6;
    }
    enum DeviceModel{
    DUMMY_MODEL = 0;
    ED527 = 1;
    ED588 = 2 ;
    JR3000 = 3;
    OPTOE = 4;
    BASLER = 5;
    SPiiPlusEC=6;
    GT6600 = 7;
    MOTOR = 8;
    }
    enum DeviceStatus{
    OFFLINE = 0;
    ONLINE = 1;
    NOT_REACHABLE =2;
    }
    message CommandResponse{
    int32 resp_code = 1;
    string resp_message = 2;
    }
    Parameter Description
    resp_code Response code from the device API
    resp_message Response message code

    CLI Command: python denso.py GetDeviceDetails Barcode_1

    Description for CLI Command:
    • GetDeviceDetails: API called to the Denso through CLI Command
    • Barcode_1: Aliased Name for Barcode Reader

  5. GetImage

    Description for GetImage API: To Capture Single Image from the device.
    RPC Command: rpc GetImage(CaptureImageRequest) returns(stream CaptureImageResponse) {}
    Request: message CaptureImageRequest {
    string path = 1;
    int32 format = 2;
    int32 size = 3;
    int32 quality = 4;
    .genric.CommandRequest cmd_request = 5;
    }
    Parameter Description
    device_alias The unique device_alias used to refer the device
    Format The format of the image (0-Jpeg and 1-Bitmap)
    Size The size of the image.(0->Full, 1->quarter, 2->Sixteenth, 3->Sixforth)
    Quality The quality of the image.(0->Low, 1->Medium, 2->High)
    Response: message CaptureImageResponse {
    optional bytes image_data = 1;
    optional string file_name = 2;
    optional int64 file_size = 3;
    .generic.CommandResponse cmd_response = 4;
    }
     message CommandResponse {
    int32 resp_code = 1;
    string resp_description = 2;
    }
    Parameter Description
    image_data The image data
    file_name File name of the image file
    file_size The denotion of the file size
    Parameter Description
    resp_code Response code from the device API
    resp_description Description of the response code

    CLI Command: python barcode.py GetImage BarCode1 barcode.jpg 0 2 2

    Description for CLI Command:
    • Barcode_1: Alias of the Device
    • barcode.jpg User defined file name with format
    • 0: The format of the image (0-Jpeg and 1-Bitmap)
    • 2: The size of the image to be captured(0->Full, 1->quarter, 2->Sixteenth, 3->Sixforth)
    • 2: The quality of the image.(0->Low, 1->Medium, 2->High)

    Limitations of the driver

    • Compatibility: The driver may only be compatible with DM262x and DM282x versions of Cognex Barcode Reader, limiting its use with other devices.
    • Integration:Integrating the driver with other systems or software may pose challenges, especially if there are compatibility issues or dependencies.
    • Modifying Environment may affect system connection.

    Precautions to take before executing the driver

    • Make sure the Device is in connected with the station
    • Ensure to giving the valid channel number and state.

GLOSSARY

  • Driver: A software component that enables communication between the operating system and a hardware device, in this case, a Barcode Reader.
  • SDK (Software Development Kit): A set of tools, libraries, and documentation that developers use to create software applications for a specific platform or framework.
  • Command: Instruction or directive sent to a Barcode Reader through the driver to perform specific actions such as movement, I/O control, or program execution.
  • Response: Feedback or acknowledgment received from the Barcode Reader I/O after executing a command, indicating the success or failure of the operation and providing relevant information or data.
  • API (Application Programming Interface): A set of rules and protocols that allow different software applications to communicate with each other.
  • Firmware Software that is embedded into a hardware device, such as a Barcode Reader, to control its operation.
  • Size: This denotes the size of the image ex. Full, quarter and half.
  • Format: This denotes the format of the image ex. Jpeg or Bitmap.
