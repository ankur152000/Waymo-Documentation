Node-RED Execution and Flow Troubleshooting Guide

  1. Introduction

    Node-RED is a flow-based development tool where each node performs a specific task and passes the message (msg) to the next node. Below is a breakdown of flow behavior and lifecycle.

    1. Message Flow:

      • A flow begins with an input node (e.g., HTTP In, MQTT In, Inject).
      • The message flows through processing nodes (e.g., Function, Change, Switch).
      • It ends in output nodes (e.g., HTTP Response, MQTT Out, Debug).

    2. Execution Lifecycle:

      • Trigger:A flow is triggered by an event (e.g., Inject button, HTTP request).
      • Propagation:The msg object is passed from one node to the next via wires.
      • Processing:Each node processes the msg according to its logic.
      • Termination:The flow ends with a node that does not pass data further (like HTTP Response).

    3. Execution Context:

      Node-RED provides three types of data context:

      • msg: Per-message scope. Changes only apply to the current message.
      • flow: Shared across nodes within the same flow.
      • global: Accessible across all flows in the instance.

  2. Common Issues and How to Resolve Them

    1. Issue 1: Node Not Executing:

      Cause: Node conditions not met or previous node not passing data

      How To Fix

      • Use Debug nodes after each node to see if msg is passed.
      • Check wires to ensure nodes are connected properly.

    2. Issue 2: Undefined or Empty msg Properties

      Cause: Accessing properties that do not exist.

      How To Fix

      • Add checks in Function nodes:

        • if (!msg.payload) {
        node.error("Payload is missing", msg);
        return null;
        }

    3. Issue 3: Function Node Not Working

      Cause: Syntax errors or logic issues.

      How To Fix:

      • Check Node-RED logs for errors.
      • Use node.warn() and node.log() to trace.

    4. Issue 4: Inject Node Doesn’t Trigger

      How To Fix:

      • Make sure node is deployed
      • Try manually clicking the Inject button.

    5. Issue 5: Flows Not Deploying

      How To Fix:

      • Ensure there are no syntax errors in Function nodes.
      • Check if any required nodes are missing (e.g., custom nodes).

  3. Advanced Debugging Techniques

    1. Use the Debug Panel

      • Set Debug nodes to show the complete message.
      • Place Debug nodes strategically across flows.

    2. Logging in Function Nodes

      node.warn("This is a warning");
      node.log("This is a log message");

    3. Catch and Status Nodes

      • Use Catch node to capture and log errors.
      • Use Status node to monitor node status (e.g., connected, error).

  4. Flow Management

    1. Flow Tabs

      • Organize logic across multiple flows.
      • Use Link In/Out nodes to manage flow transitions.

    2. Backups

      • Always keep exported JSON copies of important flows.
      • Use version control (Git) for production flows.

    3. Flow Configuration

      • Use Environment Variables for dynamic values.
      • Use Context Storage to persist data across deploys.
      • Leverage Credentials storage for secrets like API keys.

  5. Node-RED Runtime Troubleshooting

    1. Understanding Runtime Architecture

      • Node-RED runs on Node.js.
      • Each flow is compiled to a set of JavaScript functions.
      • Nodes run sequentially, and asynchronously if needed.
      • Runtime maintains a single-threaded event loop but leverages async I/O.

    2. Accessing Logs

      • Use terminal:

        node-red-log

      • Or check logs in

        /var/log/node-red.log

    3. Restart Node-RED

      • Run Command in terminal

        sudo systemctl restart nodered

      • Checking Port Usage

        netstat -tulnp | grep :1880

    4. Performance Monitoring

      • Use top, htop, or Node-RED’s built-in metrics (if enabled).
      • Consider enabling Node-RED Flow Metrics to track throughput and delays.

  6. Debugging Custom Nodes

    • Ensure the node is installed properly using npm.
    • Restart Node-RED after installing custom nodes.
    • Check the node’s source code in ~/.node-red/node_modules/
    • Use try/catch inside the custom node function for error handling.

  7. Troubleshooting Flows

    1. Tips for Diagnosing Flow Issues

      • Use Debug Nodes generously to trace the path of msg.
      • Validate logic in Switch and Change nodes.
      • Inspect Function nodes for proper output.
      • Review flow triggers — some flows may not trigger without specific conditions.

    2. Ensuring Flow Reliability

      • Avoid hard-coding paths and values.
      • Use environment variables or configuration nodes.
      • Group common patterns into subflows for reusability.

    3. Preventing Flow Errors

      • Always check if the required properties exist in msg.
      • Wrap logic in try/catch blocks if using advanced scripting.
      • Use return null; to stop flows when needed.

    4. Handling External APIs or Inputs

      • Add Retry/Timeout mechanisms.
      • Use Status and Catch nodes to handle unexpected failures
